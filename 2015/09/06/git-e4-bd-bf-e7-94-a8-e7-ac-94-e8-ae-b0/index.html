Git使用笔记（一）

发表于 2015-09-06   |   分类于 生活

实验室项目最近进入代码实现阶段，几个人看同一份代码，修改后的代码通过qq传来传去太容易乱，所以果断启用Git。写这篇日志记录一些基本操作备忘。

首先就是，把远程的repo克隆到本地
git clone "address"//address就是repo的https地址

然后在github的对应repo的设置中添加协作者，把他们的用户名添加到list中，这样他们就和你一样具有push权限啦～
接下来就是一些基本的操作：
每次添加了文件或者有修改之后，首先要将修改添加到缓冲区：
git add "change"//change 就是你修改的文件名

然后：
git commit -m "commit"//commit是你对本次修改的说明
最后：
git push origin master//因为我的项目没有其他branch所以直接push也可以

另外可以用：
git status//查看当前缓冲区的修改情况，选择进一步commit或者取消

删除文件的话，先在本地删除，然后：
git rm "name"//name就是你要删除的文件名
接下来还是commit然后push就好啦～～～

今天学会的功能基本上已经能胜任最近的工作需要了。剩下的功能下周考完托福进一步开发，我在考虑创建一个本地的git版本库，然而并没有用。。。
